## Схема реализована на эллиптической кривой secp256k1 с параметрами:

    G - генератор (базовая точка) кривой

    n - порядок циклической подгруппы

    d - приватный ключ (случайное число от 1 до n-1)

    P - публичный ключ (точка на кривой: P = d*G)

## Генерация ключей

    Выбирается случайное число d в диапазоне [1, n-1] как приватный ключ

    Вычисляется точка P = d*G как соответствующий публичный ключ

## Процесс подписания сообщения

    Генерируется случайное число k в диапазоне [1, n-1] (одноразовый номер)

    Вычисляется точка R = k*G

    Вычисляется хеш e = H(R||P||message), где H - хеш-функция SHA-256

    Вычисляется s = k + e*d (mod n)

    Подпись представляет собой пару (R, s)

## Процесс верификации подписи

    Вычисляется хеш e = H(R||P||message)

    Проверяется равенство: sG = R + eP

    Если равенство выполняется, подпись считается действительной
